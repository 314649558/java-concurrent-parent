RPC调用流程说明
  1. 服务消费方（client） 以本地调用方式调用服务
  2. client stub 接收到调用后负责将方法，参数等封装成能够进行网络传输的消息体
  3. client stub 将接收到的消息编码并转发给服务端
  4. server stub 收到消息后解码
  5. server stub 根据解码结果并调用本地服务
  6. 本地服务执行并将结果返回给server stub
  7. server stub 将返回结果进行编码并发送给消费方
  8. client stub 接收到消息并解码
  9. 消费方(client)得到结果

  小结： RPC的目标就是将2-8这些步骤封装起来，用户无需关心这些细节。可以像调用本地方法一样即可完成远程服务调用
